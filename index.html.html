NHS Sheffield Teaching Hospitals · Illustrative Case Study

Patient Referral Workflow Automation

Situation
A fragmented, largely manual referral pathway spanning 4 clinical departments with no unified tracking or SLA visibility.
Task
Lead requirements gathering and design a TO-BE automated workflow in collaboration with clinical leads, operations, and IT.
Action
Facilitated 6 stakeholder workshops, mapped AS-IS and TO-BE processes in MS Visio, produced FRD, and oversaw implementation sprints.
Result
30% increase in process transparency, measurable reduction in manual hand-off errors, and improved SLA compliance tracking.
What I delivered
  • AS-IS / TO-BE process maps documenting the full referral lifecycle across 4 departments
  • Functional Requirements Document (FRD) covering 40+ requirements with acceptance criteria
  • Stakeholder RACI matrix and communication plan for 12 stakeholders
  • Post-implementation review report with KPI baseline vs. improved metrics
Tools & methods

MS Visio (process mapping) · Power BI (tracking dashboard) · Excel (data analysis) · Agile sprint facilitation · Workshop design

NHS Sheffield Teaching Hospitals · Illustrative Case Study

SLA Performance Dashboard Suite

Situation
Clinical and operational teams had no single source of truth for SLA performance — decisions were based on outdated weekly exports.
Task
Design, build, and maintain a suite of dashboards enabling real-time KPI tracking for patient services leadership.
Action
Gathered requirements from 8 stakeholders, defined 15 KPIs, built 5 Power BI dashboards pulling from SQL data sources, with automated refresh.
Result
SLA performance improved 15% within 2 quarters; leadership cited dashboards as enabling faster, more confident decisions.
What I delivered
  • KPI framework document defining 15 metrics with owners, thresholds, and data sources
  • 5 Power BI dashboards: SLA overview, department-level, trend analysis, exception alerts, executive summary
  • SQL queries for data extraction and validation across 3 integrated systems
  • Dashboard user guide and training delivered to 20+ clinical managers
Tools & methods

Power BI · SQL · Excel · KPI framework design · Stakeholder requirements interviews · Data validation

Capgemini — Banking Client (UK) · Illustrative Case Study

Digital Banking Transformation — BRD & UAT

Situation
A UK retail bank undergoing core digital transformation with misaligned requirements between UK business teams and India-based development.
Task
Bridge the gap as BA: own end-to-end requirements documentation and UAT governance across a cross-geographical team.
Action
Authored BRS, BRD, FRD, and 200+ test cases. Conducted gap analysis in each sprint, led UAT cycles, and used SQL for data reconciliation.
Result
Post-release defects reduced 25%. Sprint predictability improved 20%. Requirements traceability achieved from business need through UAT sign-off.
What I delivered
  • Business Requirements Specification (BRS) covering 3 workstreams
  • Business Requirements Document (BRD) and Functional Requirements Document (FRD)
  • 200+ detailed test cases with pass/fail criteria and defect log integration in Jira
  • Gap analysis and impact assessment reports for each sprint validation cycle
  • SQL data validation scripts for reconciliation across integrated banking systems
Tools & methods

Jira · Confluence · SQL · Agile/Scrum · BRD/FRD authoring · UAT oversight · API regression testing

NHS Sheffield Teaching Hospitals · Illustrative Case Study

Root Cause Analysis — Service Quality Initiative

Situation
Recurring service issues flagged in patient services with no structured framework to identify root causes or track resolution.
Task
Design and execute a structured RCA programme across 100+ logged issues, categorise patterns, and deliver actionable recommendations.
Action
Applied fishbone diagrams and 5-Why frameworks. Used SQL and Excel to validate data accuracy and identify systemic vs. one-off failures.
Result
Identified 3 systemic failure categories, improved data accuracy, and delivered a prioritised improvement roadmap adopted by operations leadership.
What I delivered
  • RCA framework document with fishbone templates and 5-Why methodology guide
  • Issue categorisation report across 100+ service incidents
  • SQL-validated data accuracy audit with findings report
  • Prioritised improvement roadmap with quick wins vs. long-term fixes
Tools & methods

Root cause analysis (fishbone / 5-Why) · SQL · Excel · Data validation · Continuous improvement frameworks
